# PostSH

面向 PC 与移动端的**异步交互式 SSH 客户端**，在弱网、不稳定网络下仍可顺畅输入命令并收回显。

## 项目简介

PostSH 是一款 SSH 客户端，可运行在 PC 与手机上。与传统 SSH 客户端相比，终端界面采用**异步交互**形式：

| 对比项     | 传统 SSH 客户端           | PostSH                                                |
|------------|----------------------------|--------------------------------------------------------|
| 交互方式   | 实时双向流，强依赖长连接   | 命令本地排队，连接可用时发送，执行后回显               |
| 弱网表现   | 输入卡顿、易断线、体验差   | 可继续输入、排队发送；断线自动重连、未发命令重放       |
| 适用场景   | 稳定内网 / 机房            | 地铁、高铁、弱 Wi-Fi、高延迟/丢包网络                  |

部署细节将在此文档的「[部署说明](#部署说明)」章节维护。

## 核心特性

- **异步交互**：命令本地排队，弱网下可继续输入，连接恢复后再发送，不依赖实时双向流。
- **弱网友好**：网络较差或不稳定时仍可操作；断线自动重连，未发送命令自动重放。
- **会话恢复**：断线检测、指数退避重连；支持 tmux/screen 恢复建议，协助恢复同一 Shell。
- **连接管理**：保存主机配置，密码/密钥认证，快速连接。
- **智能补全**：基于纯本地缓存，支持命令、路径与历史补全（静态列表 + 历史记录 + 回显解析）。
- **多端支持**：支持 PC 与手机端使用。

## 技术架构

整体为**直接 SSH 连接**：客户端 ↔ SSH 服务器（目标主机），无中继服务。异步数据流：命令本地排队 → 连接可用时发送 → 执行后回显；断线时保留队列，重连后重放。

技术栈：Flutter + dartssh2。详见 [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)。会话恢复机制见 [docs/SESSION_RECOVERY.md](docs/SESSION_RECOVERY.md)。界面与补全交互见 [docs/UI_DESIGN.md](docs/UI_DESIGN.md)。

## 快速开始

**前置依赖**：Flutter 3.8+

```bash
# 克隆后进入项目目录
cd postsh

# 安装依赖
flutter pub get

# 运行（macOS 优先）
flutter run -d macos
```

开发计划与阶段划分见 [docs/DEVELOPMENT_PLAN.md](docs/DEVELOPMENT_PLAN.md)。
